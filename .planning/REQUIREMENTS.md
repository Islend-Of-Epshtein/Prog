## Функциональные требования

### FR-001: Компилируемость проекта (P0)
**Описание:** Проект должен успешно компилироваться без ошибок.

**Критерии приёмки:**
- [ ] `mvn compile` завершается успешно
- [ ] Все Java-файлы имеют корректное расширение `.java`
- [ ] Все ссылки на классы разрешаются

**Связанные проблемы:**
- CONCERNS.md:1.1 — несуществующие классы Task2Controller/Task2Dispatcher
- CONCERNS.md:1.2 — файл ProcessData.Java с неверным расширением

---

### FR-002: Работоспособность Task1 — Файловый сервер (P1)
**Описание:** Клиент может подключаться к серверу и просматривать директории/файлы.

**Критерии приёмки:**
- [ ] Сервер запускается на указанном порту
- [ ] Клиент подключается к серверу
- [ ] Сервер возвращает список файлов директории
- [ ] Сервер возвращает содержимое текстового файла

---

### FR-003: Работоспособность Task2 — Контроллер процесса (P1)
**Описание:** Контроллер генерирует данные, диспетчер отображает их в консоли.

**Критерии приёмки:**
- [ ] Controller генерирует данные температуры/давления
- [ ] Dispatcher подключается и получает данные
- [ ] Данные корректно сериализуются/десериализуются

---

### FR-004: Работоспособность Task3 — Система мониторинга (P1)
**Описание:** GUI-приложение отображает состояния установок.

**Критерии приёмки:**
- [ ] Controller запускается как отдельный процесс
- [ ] Dispatcher подключается и отображает состояния
- [ ] GUI корректно отображает статусы (РАБОТАЕТ/АВАРИЯ/РЕМОНТ)

## Нефункциональные требования

### NFR-001: Безопасность портов (P1)
**Описание:** Валидация портов в диапазоне 1024-65535.

**Критерии приёмки:**
- [ ] Порты < 1024 блокируются (требуют привилегий)
- [ ] Порты > 65535 блокируются
- [ ] Показывается понятное сообщение об ошибке

**Связанные файлы:**
- Task3/Dispatcher.java:121-128
- Task3/Controller.java:176-179

---

### NFR-002: Безопасная сериализация (P1)
**Описание:** Защита от уязвимостей Java-сериализации.

**Критерии приёмки:**
- [ ] Валидация десериализуемых объектов
- [ ] Ограничение размера сериализованных данных
- [ ] Рассмотрение альтернативных форматов (JSON)

**Связанные файлы:**
- Base/Server.java, Base/Client.java

---

### NFR-003: Корректная обработка ресурсов (P1)
**Описание:** Все ресурсы корректно закрываются при ошибках.

**Критерии приёмки:**
- [ ] try-with-resources для всех потоков
- [ ] Нет утечек сокетов
- [ ] Корректное завершение потоков

---

### NFR-004: Потокобезопасность (P1)
**Описание:** Корректная работа в многопоточной среде.

**Критерии приёмки:**
- [ ] synchronized или Atomic* для общих переменных
- [ ] volatile для флагов завершения
- [ ] Корректная обработка InterruptedException

## Требования к качеству кода

### QR-001: Соответствие Java Naming Conventions (P2)
**Описание:** Код следует стандартам именования Java.

**Критерии приёмки:**
- [ ] Классы: PascalCase
- [ ] Методы: camelCase
- [ ] Пакеты: lowercase
- [ ] Константы: UPPER_SNAKE_CASE

**Текущие нарушения:**
- Методы `Write()`, `Read()`, `Accept()` — должны быть `write()`, `read()`
- Пакеты `Base`, `GUI`, `Task1` — должны быть `base`, `gui`, `task1`

---

### QR-002: Современное тестирование (P2)
**Описание:** Использование JUnit 5 вместо JUnit 3.8.1.

**Критерии приёмки:**
- [ ] JUnit 5 Jupiter в зависимостях
- [ ] Минимум 1 unit-тест на каждый пакет
- [ ] Интеграционные тесты для сокетов

---

### QR-003: Документирование кода (P3)
**Описание:** JavaDoc для публичных API.

**Критерии приёмки:**
- [ ] JavaDoc для публичных классов
- [ ] JavaDoc для публичных методов
- [ ] Описание параметров и возвращаемых значений
